# 스크린샷 검증 가이드

`make_screenshots.py` 실행 후 생성된 PNG들을 검증한다.

---

## 검증 스크립트: verify_screenshots.py

전체 검증 항목을 자동으로 수행하고 결과 리포트를 출력한다.

언어 수는 `output/` 폴더를 자동 스캔해서 감지한다 (`discover_languages()` 함수).

---

## 검증 항목

### 1. 파일 존재 검증

```
[검증 1] 폴더/파일 존재 확인
- screenshots/ 폴더 존재
- output/ 의 모든 언어 폴더에 대응하는 screenshots/ 하위 폴더 존재
- 각 폴더에 정확히 6개 PNG 파일 존재
- 총 N × 6 개 파일

[기대 파일명]
{번호}_{언어코드}_iphone_1.png
{번호}_{언어코드}_iphone_2.png
{번호}_{언어코드}_iphone_3.png
{번호}_{언어코드}_ipad_1.png
{번호}_{언어코드}_ipad_2.png
{번호}_{언어코드}_ipad_3.png

[FAIL 조건]
- 폴더 누락
- 파일 누락
- 파일명 규칙 불일치
```

### 2. 파일 포맷 검증

```
[검증 2] PNG 포맷 확인
- 모든 파일이 PNG 포맷인지 확인 (확장자 + magic byte)
- JPEG, WebP 등 다른 포맷 섞여있는지 확인

[FAIL 조건]
- PNG가 아닌 파일 발견
- 손상된 PNG 파일
```

### 3. 픽셀 크기 검증

```
[검증 3] 픽셀 크기 확인
- iPhone: 정확히 1242 x 2688
- iPad: 정확히 2048 x 2732

[FAIL 조건]
- 1픽셀이라도 크기 다름 (App Store 규격 위반)
```

### 4. 빈 파일 검증

```
[검증 4] 0바이트 또는 비정상 작은 파일 검출
- 50KB 이하 파일은 의심
- 0바이트 파일은 무조건 FAIL

[FAIL 조건]
- 0바이트 파일
- 의심스럽게 작은 파일 (50KB 이하)
```

### 5. 색상 모드 검증

```
[검증 5] PNG 색상 모드
- RGB 또는 RGBA 허용
- 그 외(P, L, CMYK 등) FAIL

[FAIL 조건]
- 흑백/팔레트/CMYK 모드
```

### 6. 폰트 크기 로그 검증

```
[검증 6] 합성 시 발생한 에러 로그 확인
- make_screenshots.py 실행 중 50pt 이하로 떨어진 케이스
- skip된 파일 목록

[FAIL 조건]
- skip 발생 (텍스트가 너무 길어서 50pt에서도 안 들어가고 줄바꿈도 실패)
```

## 검증 출력 형식

```
================================================================
[검증 시작] screenshots/ 폴더
================================================================

[검증 1] 파일 존재
  ✓ N개 폴더 모두 존재
  ✓ N×6개 PNG 파일 모두 존재
  ✓ 파일명 규칙 모두 준수

[검증 2] PNG 포맷
  ✓ N×6/N×6 PNG 정상

[검증 3] 픽셀 크기
  ✓ iPhone (1242x2688): N×3/N×3 정상
  ✓ iPad (2048x2732): N×3/N×3 정상

[검증 4] 빈 파일
  ✓ 의심 파일 없음

[검증 5] 색상 모드
  ✓ N×6/N×6 RGB/RGBA

[검증 6] 합성 로그
  ✓ skip 발생 없음

================================================================
[검증 요약]
================================================================
- 총 파일: N×6 개
- 정상: N×6 개
- 실패: 0 개

[전체 결과]: PASS
================================================================
```

## FAIL 발생 시 처리

```
================================================================
[검증 실패 발견]
================================================================

[검증 3 - 픽셀 크기 불일치]
- screenshots/N_xx/N_xx_iphone_2.png
  기대: 1242x2688
  실제: 1240x2688
  → 1픽셀 누락. 합성 코드 확인 필요.

[검증 6 - skip 파일]
- N_xx/N_xx_2.txt
  → 50pt에서도 텍스트가 Safe Zone 초과
  → 자동 줄바꿈 시도 후에도 실패
  → 해결 방법:
    1) 원본 텍스트(output/N_xx/N_xx_2.txt) 줄여서 재번역
    2) 또는 Safe Zone 좌우 마진을 더 줄임 (mockup 침범 위험 확인)

[전체 결과]: FAIL
================================================================
```

---

## 자동 수정은 하지 않음

검증 스크립트는 **문제만 발견하고 보고**하며, 자동 수정은 하지 않는다.

문제 발견 시:
1. 원인 분석 (텍스트 길이? 폰트 누락? 코드 버그?)
2. 사용자가 결정 후 수동으로 재실행

---

## 시각적 확인 (수동)

자동 검증 외에 **샘플 육안 확인**을 권장한다:

```
권장 샘플:
- 라틴 알파벳 (en, de, es 등 1개)
- CJK (ja, zh_Hans, zh_Hant 중 1~2개)
- RTL (ar, he 중 1개)
- 인도계 (hi, ta 등 사용 시 1개)
- 그 외 비기본 문자 (th, ka 등 사용 시)
```

각 샘플에서 확인할 점:
- 글자가 깨지지 않는가? (폰트 매핑 누락 시 □□□□로 표시됨)
- 글자가 mockup 위로 침범하지 않는가?
- 그림자가 자연스럽게 우측 하단에 보이는가?
- 두 줄이 수직 중앙에 잘 배치되어 있는가?
- 폰트 크기가 너무 작거나 크지 않은가?
- RTL 언어의 글자 연결이 정상인가?
